THE BIAS
Introduction

Here, we additionally introduce BIAS to the NEURON. In other words, the mind will have two PARAMETERS: WEIGHT and BIAS.

During TRAINING, at every LESSON, the PARAMETERS will compete over which gets updated more. The PARAMETER that has more INFLUENCE on OUTPUT receives a larger update. To see this competition more clearly, let's set SMALL to 1.0:

SMALL = 1.0
The Mind

We want to train a machine mind to convert Celsius temperatures into Fahrenheit temperatures. In this scenario, INPUT represents Celsius and OUTPUT represents Fahrenheit.

This mind will be called the Converter °C to °F.

As previously noted, it will have two PARAMETERS: WEIGHT and BIAS.

The mind will be just a formula like that:

OUTPUT = INPUT × WEIGHT + BIAS
Initial PARAMETERS

Suppose we don't yet know what these PARAMETERS should be, so let's start by setting them to zero:

WEIGHT = 0.0
BIAS = 0.0

At first, the mind will generate nonsense.

The Dataset

We have the following training DATASET:

# INPUT TARGET
EXAMPLE 1 0.0 32.0
EXAMPLE 2 1.0 33.8

EXAMPLE 1 means when INPUT is 0.0, the OUTPUT must be 32.0. In other words, 0.0 degrees Celsius equals 32.0 degrees Fahrenheit.

EXAMPLE 2 means when INPUT is 1.0, the OUTPUT must be 33.8. In other words, 1.0 degree Celsius equals 33.8 degrees Fahrenheit.

During TRAINING, at every LESSON, the mind will update every PARAMETER so that the OUTPUT gets closer to the TARGET.

The Update Rule

The mind will update every PARAMETER using the Fundamental Update Rule:

Fundamental Update Rule
PARAMETER = PARAMETER + PARAMETER_update
PARAMETER_update = ERROR × PARAMETER_OUTPUT_influence × SMALL

Because the mind has two PARAMETERS — WEIGHT and BIAS — we obtain:

Fundamental Update Rule for WEIGHT
WEIGHT = WEIGHT + WEIGHT_update
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL
Fundamental Update Rule for BIAS
BIAS = BIAS + BIAS_update
BIAS_update = ERROR × BIAS_OUTPUT_influence × SMALL

Don't worry — it's really simple. Just follow the calculations below.

Testing

To test the mind, we give it a problem — an EXAMPLE — it has never encountered before.

WEIGHT = 1.8
BIAS = 32.0

EXAMPLE FOR TESTING:

INPUT = 10.0
TARGET = 50.0
Forward Pass
PRODUCT = INPUT × WEIGHT = 10.0 × 1.8 = 18.0
SUM = PRODUCT + BIAS = 18.0 + 32.0 = 50.0
OUTPUT = SUM = 50.0
Error
ERROR = TARGET − OUTPUT = 50.0 − 50.0 = 0.0

This mind — since ERROR = 0.0 — works perfectly!

It has learned the rule: convert Celsius temperatures into Fahrenheit.

The Converter °C to °F can now solve problems it has never encountered before.

The mind is now just a formula:

OUTPUT = INPUT × 1.8 + 32.0

Note

Machines improve themselves by learning from their own errors. They do this using the Fundamental Update Rule:

Fundamental Update Rule
PARAMETER = PARAMETER + PARAMETER_update
PARAMETER_update = ERROR × PARAMETER_OUTPUT_influence × SMALL

PARAMETER → This is the thing the machine is trying to get right.

PARAMETER_update → How much PARAMETER is updated.

ERROR → How wrong the machine was.

PARAMETER_OUTPUT_influence → INFLUENCE of PARAMETER on OUTPUT.

SMALL → A small number that keeps each update from being too big.


During TRAINING, at every LESSON, the PARAMETERS compete over which gets updated more. The PARAMETER that has more INFLUENCE on OUTPUT receives a larger update.
