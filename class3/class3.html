Class 3
SMALL
Introduction

The SMALL is just a small number.

It handles big numbers in a dataset, keeping each update from being too big.

For example, let's set it to 0.0100:

SMALL = 0.0100
The Mind

We want to train a machine mind that may learn to multiply any number by 5.0000.

The mind will be called the Multiplier By Five.

It will have a single PARAMETER: WEIGHT.

Initial PARAMETER

Suppose we don't yet know what the PARAMETER should be, so let's start by setting it to zero:

WEIGHT = 0.0000

At first, the mind will generate nonsense.

The Dataset

We have the training DATASET consisting of two EXAMPLES:

INPUT TARGET
EXAMPLE 1  0.6000  3.0000
EXAMPLE 2 -10.0000 -50.0000

EXAMPLE 1 means when the INPUT is 0.6000, the OUTPUT must be 3.0000.
EXAMPLE 2 means when the INPUT is -10.0000, the OUTPUT must be -50.0000.

During training, at every LESSON, the mind will update the PARAMETER so that the OUTPUT gets closer to the TARGET.

The Update Rule

The mind will update the PARAMETER using the Fundamental Update Rule:

Fundamental Update Rule
PARAMETER = PARAMETER + PARAMETER_update
PARAMETER_update = ERROR × PARAMETER_OUTPUT_influence × SMALL

Because the mind has just one PARAMETER — WEIGHT — we obtain:

Fundamental Update Rule for WEIGHT
WEIGHT = WEIGHT + WEIGHT_update
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL

Don't worry — it's really simple. Just follow the calculations below.

Training
LESSON 1
WEIGHT = 0.0000
SMALL = 0.0100

EXAMPLE 1:

INPUT = 0.6000
TARGET = 3.0000
Forward Pass
PRODUCT = INPUT × WEIGHT = 0.6000 × 0.0000 = 0.0000
OUTPUT = PRODUCT = 0.0000
Error
ERROR = TARGET − OUTPUT = 3.0000 − 0.0000 = 3.0000
Backward Pass — Backpropagation
Universal Update Rule for WEIGHT
WEIGHT = WEIGHT + WEIGHT_update
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL
WEIGHT = 0.0000
ERROR = 3.0000
What is WEIGHT_OUTPUT_influence here?
Our formula is: OUTPUT = INPUT × WEIGHT
The number in front of WEIGHT is INPUT.
Therefore: WEIGHT_OUTPUT_influence = INPUT
WEIGHT_OUTPUT_influence = INPUT = 0.6000
SMALL = 0.0100
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL = 3.0000 × 0.6000 × 0.0100 = 0.0180
WEIGHT = WEIGHT + WEIGHT_update = 0.0000 + 0.0180 = 0.0180
WEIGHT = 0.0180
What just happened?

The mind has just improved itself.

Now, if it receives the same INPUT of 0.6000, the OUTPUT becomes 0.0108 — which is closer to the TARGET (3.0000) than the previous OUTPUT (0.0000) was.

The ERROR becomes smaller:

ERROR = 2.9892


LESSON 1 ends here.

LESSON 2
WEIGHT = 0.0180
SMALL = 0.0100

EXAMPLE 2:

INPUT = -10.0000
TARGET = -50.0000
Forward Pass
PRODUCT = INPUT × WEIGHT = -10.0000 × 0.0180 = -0.1800
OUTPUT = PRODUCT = -0.1800
Error
ERROR = TARGET − OUTPUT = -50.0000 − (−0.1800) = -49.8200
Backward Pass — Backpropagation
Universal Update Rule for WEIGHT
WEIGHT = WEIGHT + WEIGHT_update
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL
WEIGHT = 0.0180
ERROR = −49.8200
What is WEIGHT_OUTPUT_influence here?
Our formula is: OUTPUT = INPUT × WEIGHT
The number in front of WEIGHT is INPUT.
Therefore: WEIGHT_OUTPUT_influence = INPUT
WEIGHT_OUTPUT_influence = INPUT = −10.0000
SMALL = 0.0100
WEIGHT_update = ERROR × WEIGHT_OUTPUT_influence × SMALL = −49.8200 × (−10.0000) × 0.0100 = 4.9820
WEIGHT = WEIGHT + WEIGHT_update = 0.0180 + 4.9820 = 5.0000
WEIGHT = 5.0000
What just happened?

The mind has just improved itself.

Now, if it receives the same INPUT of −10.0000, the OUTPUT becomes −50.0000 — which is closer to the TARGET (−50.0000) than the previous OUTPUT (-0.1800) was.

Wait — it's not just closer. It's exact!

The OUTPUT is now -50.0000 — exactly the same as the TARGET.

The ERROR becomes zero:

ERROR = 0.0000


LESSON 2 ends here.

Testing

To test the mind, we give it a problem — an EXAMPLE — it has never encountered before.

WEIGHT = 5.0000

EXAMPLE FOR TESTING:

INPUT = 10.0000
TARGET = 50.0000
Forward Pass
PRODUCT = INPUT × WEIGHT = 10.0000 × 5.0000 = 50.0
OUTPUT = PRODUCT = 50.0000
Error
ERROR = TARGET − OUTPUT = 50.0000 − 50.0000 = 0.0000

This mind — since ERROR = 0.0000 — works perfectly!

It has learned the rule: multiply any number by 5.0.

The Multiplier By Five can solve problems it has never encountered before.

Note

Machines improve themselves by learning from their own errors. They do this using the Fundamental Update Rule:

Fundamental Update Rule
PARAMETER = PARAMETER + PARAMETER_update
PARAMETER_update = ERROR × PARAMETER_OUTPUT_influence × SMALL

PARAMETER → This is the thing the machine is trying to get right.

PARAMETER_update → How much PARAMETER is updated.

ERROR → How wrong the machine was.

PARAMETER_OUTPUT_influence → INFLUENCE of PARAMETER on OUTPUT.

SMALL → A small number that keeps each update from being too big.
